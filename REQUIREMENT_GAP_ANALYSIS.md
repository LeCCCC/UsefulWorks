# 进程控制和调度需求符合性评估

结论：**当前项目“部分满足”题目要求，但尚不能视为完整达标**。

## 已满足点

1. **具备 PCB 模型与进程基础属性**：包括进程名、优先级、到达时间、总运行时间、已用 CPU 时间、状态、开始/结束时间与周转时间。  
2. **实现两类调度器**：
   - 基于优先级的时间片轮转（PriorityRR）
   - 三级反馈队列轮转（MLFQ3）
3. **支持可视化图形界面（Swing）动态展示**：
   - 单步推进与自动运行
   - 显示时间、当前运行进程、队列、日志、进程表
   - 可切换两种算法
4. **可计算平均周转时间**：在所有进程完成后显示平均周转时间。

## 未满足或存在偏差的点

1. **进程控制“阻塞/唤醒/撤销”未在 GUI 中可操作**：
   虽然引擎有 `block/wake/kill` 方法，但界面没有对应按钮与交互流程，无法完成题目强调的进程控制过程演示。
2. **死锁检测/死锁退出未实现**：
   题目要求“系统处于死锁或所有进程完成时退出”，当前仅有“全部完成”判断。
3. **PriorityRR 抢占逻辑对“时间跨越到达事件”处理不完整**：
   调度步长可能一次推进多个时间单位，代码只对 `arrivals.getOrDefault(now, ...)` 进行抢占检查，可能漏掉时间片中间到达进程触发抢占的情况。
4. **日志中优先级衰减记录重复追加**：
   `step()` 中存在重复 `pri+=decay` 日志拼接，和实际行为不一致。
5. **工程构建配置异常**：
   `pom.xml` 指向 `spring-boot-starter-parent:4.0.3`，当前环境解析失败（403），导致无法正常 `mvn test` 验证。

## 综合建议

若要达到课程题目“完整实现”标准，建议优先补齐：

1. GUI 增加阻塞/唤醒/撤销操作入口，并在日志与状态区实时展示状态迁移；
2. 引擎加入死锁检测策略与终止条件；
3. 修正到达-抢占时序（按时间轴处理到达事件，而非仅当前 `now`）；
4. 修复重复日志；
5. 修正 `pom.xml` 父版本到可解析的 Spring Boot 版本，并补充单元测试/调度用例。
